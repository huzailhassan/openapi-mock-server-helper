Schema 规范

每个 Schema 由三部分组成：TypeDefs、Resolvers、REST 端点。保存后立即生效。

1. TypeDefs — GraphQL SDL

extend type Query {
  balance(userName: String!, currency: String!): WalletResult
}

extend type Mutation {
  bet(transactionId: String!, userName: String!, currency: String!, betAmount: Float!): WalletResult
  payout(transactionId: String!, userName: String!, currency: String!, payAmount: Float!): WalletResult
}

type WalletResult {
  userName: String
  currency: String
  balance: Float
  error: String
}

2. Resolvers — JS 函数体

const cache = new Map();

async function getOrCreate(userName) {
  let user = await prisma.user.findUnique({ where: { username: userName } });
  if (!user) user = await prisma.user.create({
    data: { username: userName, balance: 0, environment: 'prod', userId: BigInt(Date.now()) }
  });
  return user;
}

return {
  Query: {
    balance: async (_parent, { userName, currency }) => {
      const user = await getOrCreate(userName);
      return { userName, currency, balance: Number(user.balance), error: null };
    }
  },
  Mutation: {
    bet: async (_parent, { transactionId, userName, currency, betAmount }) => {
      if (cache.has(transactionId)) return cache.get(transactionId);
      const user = await getOrCreate(userName);
      if (Number(user.balance) < betAmount)
        return { userName, currency, balance: Number(user.balance), error: 'insufficient balance' };
      const updated = await prisma.user.update({
        where: { username: userName },
        data: { balance: { decrement: betAmount } }
      });
      const result = { userName, currency, balance: Number(updated.balance), error: null };
      cache.set(transactionId, result);
      return result;
    }
  }
};

3. REST 端点配置（JSON）

每个端点定义请求参数、GraphQL query、字段映射。系统自动生成 POST /rpc/{schemaName}/{operation} 路由。

字段说明
required必填的请求体字段（除 requestId/requestAt 之外）
optional选填字段
query执行的 GraphQL query/mutation 字符串
rootField从 GraphQL 结果里取哪个字段作为 data
mappingGraphQL 变量名 → 请求体字段名的映射
errorCodesGraphQL error 字符串 → REST errorCode 的映射
